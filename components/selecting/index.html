A common task where a user is able to grab certain items such as text, images, or other medias that are on the computer. It is a quick way for the computer to know what it is that the user is focusing on.

Selecting is usually seen as blue highlights for text and a white half-opaque rim around a file icon.

Typical Behavior how it usually behaves

When a user selects, the most typical appearance is that the item itself highlights in a different color. However for different items the selection will look different.

For text, the cursor turns into a text cursor as the mouse hovers over. When the user clicks and drags the mouse, it is then that the choosen characters highlight in a different color.

For items such as folders and files, when the item is clicked the a rim halo appears around the folder to signify that the computer is currently focusing on that particular object. It is from there that the user can choose to open it and do other commands with it.

Events

Text selection : When text-selecting the major actions is clicking, dragging, and letting go.

  • Click: The click signifies the beginning of the selection. As the user is hovering over the text, the mouse cursor will then turn into a text cursor. That is usually the sign that a user is able to do a text selection. Another event that occurs due to clicking is after the user has made their selection if they click without holding down the command or shift keys they will unselet that particular item.
  • Dragging: From the starting point, the user drags along the text to the last point of where they would like to select. Whereever the mouse hovers during this dragging state is what is selected.
  • Letting go: When the user reaches the end of their selection, by letting go they signify that that is the end of what they choose. At this point the cursor now turns back into a normal mouse cursor and the text that they would like to select is in blue. It is from this step that the user can do whatever action they would like with the text, whether that is copying it to the clipboard etc.

File selection: File selecting is very similar to the text selection except the major event is clicking.

  • Click: When the user clicks on the icon that they would like, that is when the item highlights and it signifies that it is selected. For most computers, when holding down shift and command keys, more items can be selected. When a file is seleected, if a user clicks on an area that is not another file or the same sort of icon, it will deselect the selection.

Variants how it varies per location

Though the act of selecting is very similar, how it displays can be different depending on the user preference. In a computer interface, selecting is done primarily with clicking and dragging. Meanwhile for touch devices, selecting is done primarily with taps and holds.

Depending on the user preferences, selecting's apperance may change to different colors.

Priority Metrics Selecting and its Usability Metrics

Selecting is something in which learnability, memorability, as well as errors are very important metrics. Users use selecting to do other functions and being able to quickly select to go to the next action is very important. Being able to learn how to select quickly as well as efficiently is important. Also because selecting is a prior action needed for other commands such as copying or moving files, the quicker it is to get selection done is also very important (efficiency). Also if the user has not done this in a while this information should be easily recalled (memorability). Errors should also be very low, for if there are any errors in selection it could lead to dire consequences such as deleting files or overwriting data.

The only metric that is not as important is satisfaction. This is because selecting is a much more utilitarian action and component and the user's enjoyment does not vary with selecing. The user could get annoyed if selecting is not memorable or efficient but the act of selecting does not need satsifaction for it to be a great component.

Key Chararacteristics What do all great selections have?

  • Prevent Errors: Users should be forgiven if they select the wrong items. They should be able to deselect objects that they no longer need or add more selections.
  • Feedback: When trying to select, the user should get immediate feedback as to what he or she is selecting. Also they should know if what they are tapping/clicking is selectable. That way the user's mental model of selectable items will be consistent to the designer's mental model.
  • Consistency: The actions that are required for selecting a certain type of object should be consistent that way selecting will have a higher level of memorability as well as efficiency. If they were different then it will confuse the user and complicate their mental model of selecting.
  • Shortcuts: To help with the efficiency of selecting, shortcuts should be added to allow users to select a large amount of objects at once. For example, control+A is a common shortcut that is used to select all of the text.

iOS iPhone4s

For iOS, selecting is a little tricker. Unlike on a laptop where a user is able to click and drag, the user must use a combinations of different taps, holds, and drags in order to select different items. Also for iOS the only selection items are texts and images. Unlike computers, there are no files, instead they are apps that one must tap to open.

Appearance

User clicks and holds down in front of text and a magnifying like glass appears showing where the text selector is.

After releasing hold, an option for Selecting and other options appear, as seen in the image below.

After tapping on select, the user is able to tap and drag either ends of the selecting blue vertical bars and move them around to make their selection

When there is no more action, a dialog menu pops up asking for what action the user would like to do next.

Though the actions are different, the events as well as the Priority metrics are the same.

Also very similary to the Human Interface Guidelines for OSX, the iOS guidelines highlight similar key characteristics:

  • Feedback: With the use of slight highlights whenever a user taps or uses gestures
  • Direct Manipulation
  • User Control
